## The ONE confirmed finding

**[MEDIUM→LOW] Entropy stride-grid origin mis-phase in `live_trade.py` (deployment fidelity, NOT a leak).**

- `features.py:104-106` anchors the sample-entropy stride grid at the **absolute series origin**
  (`(abs_start + i - W) % stride == 0`), then `:107` `ffill()`s between grid points.
- Training/validation/proof all share the **2009-08-01** origin: `footer.py.tmpl:529/533` call
  `build_feats` with **no** `abs_start` (defaults 0); `infer_online.py.tmpl:16` starts 2009-08-01 and
  `:114` computes `abs_start = len(blc) - len(lc)` (the true 2009-relative offset); `verify.py` likewise.
  So the OOS online proof is **valid** — and that is why the gating path is clean.
- But `live_trade.py.tmpl:76` starts **2023-08-01** and `:200` computes `abs_start` from a ~2023 warm
  origin (`history(220d)`). The entropy grid is phase-shifted by `Δ(2009→2023) mod stride{10,20,40}`, so
  the **frozen** model is fed an entropy value taken at a different grid phase (a window ending up to
  `stride−1` bars away) than it was trained on.
- **Impact:** live-vs-backtest signal fidelity only, bounded to the 8 ffill-smoothed entropy features, and
  only if they survive `reduce_dims` into `kept_idx`. **No future information is used** (both phases are
  causal trailing windows). It **cannot** move the `infer.py` gating Calmar. The `infer_online` proof
  provably does **not** transfer to `live_trade` because they use different origins — a concrete example
  of why "replay-proven" ≠ "live-ready" (gap T36 below).
- **Fix:** anchor `live_trade`'s `abs_start` to a 2009-equivalent offset (e.g. carry the training origin's
  bar count, or make the entropy grid origin-independent / phase-free).

## Blind-spot GAPS (no active leak today, ranked by leverage)

1. **Adjusted-price / point-in-time (T22) — INVESTIGATED 2026-06-07; recommendation pending a flip.**
   *Verified:* no `set_data_normalization` mode is set in **any** template (footer/infer/infer_online/
   verify/live_trade/portfolio/benchmark), so `add_equity` uses QC's default **`Adjusted`** mode. Per QC
   docs, Adjusted/SplitAdjusted/TotalReturn *"use the entire split and dividend history to adjust
   historical prices … regardless of the backtest end date"* — i.e. a price at sim-time `t` embeds
   corporate actions with ex-dates **after** `t`: a genuine **level look-ahead**, and it differs
   backtest-vs-live (live can only adjust up to "now"). **BUT** this pipeline consumes scale-invariant
   transforms (log-returns + ratio-based bar thresholds), so the global future factor **cancels** — the
   leak survives only at the handful of ex-dividend/split bars per year. Materiality: **GLD = exactly zero**
   (no splits/divs); UUP/IWM/SPY negligible (~4 ex-div bars/yr); **USO-family the only notable case**
   (2020 reverse split). Sources: `quantconnect.com/docs/v2/writing-algorithms/securities/asset-classes/
   us-equity/requesting-data` (entire-history back-adjustment + `ScaledRaw` point-in-time mode + API),
   `.../corporate-actions` (live vs backtest timing; dividends credited as cash under Raw).
   **Recommendation:** set `data_normalization_mode=DataNormalizationMode.RAW` (or `ScaledRaw` for
   gap-free point-in-time) explicitly at **every** `add_equity`, consistently across all templates so
   train/infer/online/verify/live agree. Raw is point-in-time-correct and live-matching; PnL is preserved
   (dividends → cash, splits → share-qty). **Not applied here** — it changes historical results (GLD
   unaffected; USO-family needs explicit re-check) and so requires a full re-validation pass via the
   driver, which cannot run in this session. Awaiting the go/keep decision.
2. **Multi-param non-`BUILDER_CLASSES` axes are online-unverified (C14/C32).** entropy, fracdiff, diurnal,
   kalman, newma carry frozen fitted params, are excluded from `BUILDER_CLASSES`, and skip `verify.py` +
   `infer_online` byte-exact replay. Not deployable-online-safely. *Mitigated:* no champion uses them —
   but block them from promotion until they have an online builder.
3. **`infer_online` prediction cross-check is weak (C26-C30).** strided (`_XSTRIDE=20` → ~95% of OOS pred
   bars unchecked), timestamp **inner-join** (a bar-SET divergence is invisible), pass gate is
   `n>0 & maxd<=1e-6` with **no coverage floor**, warmup bars excluded, `featlen_mismatch` non-fatal. The
   *bars* check (`verify.py`) is full-coverage and fine; it's the *prediction* check that is thin. Tighten
   before promoting any new champion (add a coverage floor, outer-join, fatal featlen).
4. **No forward/shadow live test (T36/C36).** All proofs are batch==online on **saved historical** bars;
   nothing exercises genuinely streaming unseen data (drift, feature freshness, latency, fallback). The
   confirmed finding above is the canonical reason this matters.
5. **TRAIN→VAL purge gap (C16).** `_run_cell` embargoes only the val→test boundary; `tx = fv & ly & tr_m`
   applies **no** symmetric purge at `TRAIN_END`, so the last ~`_EMBARGO` train bars carry labels computed
   from early-VAL returns and enter the fit — minor val_auc selection inflation, not an OOS-Calmar leak.
6. **None-reach segmentation labelers (C40).** `trend_scan`, `dc_trend`, `mh_consensus` declare
   `reach=None` and lean on the 200-bar floor (internal L≤80 today). Add an assertion that each
   None-reach labeler's internal max horizon ≤ 200 so a future widening can't silently out-reach embargo.
7. **Append-OOS-invariance "teeth" coverage (C13/C37).** the perturb-the-future probe runs only for
   `NEW_AXES`; deployed/fitted axes (logdollar, imbalance family, entropy, run, spectral, vpin, jump…)

8. **Multiple-testing depth (C8/C23/C24).** best-of-N + Bonferroni-PSR are binding; PBO/CSCV and N_eff

9. **Survivorship / front-of-funnel (T21).** the 311-ETF screen selects from currently-listed ETFs

10. **`verify_online_bars.py` hardcoded pairs (C39).** the standalone harness covers only GLD/IWM/TLT;

1. **Confirmed finding — live_trade entropy grid phase.** `footer` now persists `oos_start_abs` (the
   2009-origin axis-bar index of the first OOS bar) in every model bundle (metadata only — `infer*`/
   `verify` ignore it). `live_trade.py` anchors on its first live bar and reconstructs the true 2009-origin
   index so the entropy stride-grid phase matches training; the frozen model now receives the same entropy

   coverage); feature-length drift is now **fatal**; `preds_match` additionally requires full coverage of

   into early VAL. *Follow-up:* wire `__TRAINPURGE__` into the driver/orchestrator to expose it; enabling

5. **Adjusted-price → RAW (APPLIED 2026-06-07).** `data_normalization_mode=DataNormalizationMode.RAW`
